# VM Lifecycle

pelagos-mac runs a single persistent Linux VM as a background daemon.
The VM boots once and stays alive between container invocations, giving
warm-start latency of ~100 ms for subsequent commands.

---

## Auto-start

Every user-facing command (`run`, `exec`, `ping`, `ps`, `logs`, `stop`,
`rm`, `vm shell`, `vm console`, `vm ssh`) starts the daemon automatically
if it is not already running. There is no separate "start" step.

```
pelagos ping          # boots VM if needed, returns "pong"
pelagos run alpine /bin/echo hello
pelagos vm ssh        # boots VM if needed, then SSH in
```

Cold boot takes ~1–2 s. All subsequent commands reuse the running VM.

---

## Status & control

```bash
pelagos vm status     # running (pid 12345)  OR  stopped (exit 1)
pelagos vm stop       # SIGTERM the daemon; waits up to 15 s for exit
```

After `vm stop`, the next command triggers a cold boot automatically.

---

## Direct VM access (no container namespace)

Three modes for getting a shell or running commands directly in the VM:

| Command | Transport | Notes |
|---|---|---|
| `pelagos vm shell` | vsock | Busybox `/bin/sh`; fastest |
| `pelagos vm console` | hvc0 serial | Raw boot console; Ctrl-] to detach |
| `pelagos vm ssh [-- cmd]` | SSH (dropbear) | Full SSH; supports port-forwarding |

All three auto-start the daemon if needed.

Convenience wrappers (mirror `vm-shell.sh`):

```bash
scripts/vm-shell.sh
scripts/vm-console.sh
scripts/vm-ssh.sh [-- cmd args]
```

---

## State files

Stored in `~/.local/share/pelagos/` (respects `$XDG_DATA_HOME`):

| File | Purpose |
|---|---|
| `vm.pid` | Daemon PID |
| `vm.sock` | Unix socket — vsock proxy for container commands |
| `console.sock` | Unix socket — hvc0 serial relay |
| `vm.mounts` | Active virtiofs shares (JSON) |
| `vm_key` | Ed25519 private key for `vm ssh` (generated by `make image`) |
| `vm_key.pub` | Corresponding public key (baked into initramfs as root's `authorized_keys`) |

## e2e test interaction

`scripts/test-e2e.sh` stops and restarts the daemon several times during
the run (tests 6 and 13 call `vm stop` to reset mount configuration).
After the test suite completes the daemon is stopped. Run `pelagos ping`
or any other command to bring it back up.

---

## Mounts are fixed at daemon start

virtiofs shares (`-v /host/path:/container/path`) are passed at daemon
launch and cannot change while the daemon is running. Changing mounts
requires stopping the daemon first:

```bash
pelagos vm stop
pelagos -v /new/path:/data run alpine ls /data
```
